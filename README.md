# Masked NimStim Face Task

This repository runs a PsychoPy masked emotional-face task using NimStim faces. The NimStim image files are private stimulus materials and must not be committed to Git or made publicly available.

## Private Face Files

Put the private NimStim archive at the project root as:

```text
NimStim.zip
```

Then unzip it into the gitignored `resources/` folder:

```bash
mkdir -p resources
unzip -q NimStim.zip -d resources
```

After unzipping, the expected folder is:

```text
resources/NimStim/
```

You can check the location with:

```bash
test -d resources/NimStim && echo "NimStim folder found"
```

The following paths are intentionally ignored by Git:

```text
NimStim.zip
NimStim/
resources/NimStim/
stimuli_generated/
conditions/*.csv
data/
logs/
```

Do not use `git add -f` on any of those paths. The generated mirrored face files in `stimuli_generated/` are also derived face stimuli and should be treated as private.

## Setup From A Fresh Clone

Run these commands from the project root:

```bash
cd /path/to/emotional_reactivity
conda env create -p ./.conda_env -f environment.yml
mkdir -p resources
unzip -q NimStim.zip -d resources
test -d resources/NimStim
./.conda_env/bin/python build_stimuli.py --seed 20260625
```

The build step creates the local-only mirrored neutral targets and condition files:

```text
stimuli_generated/*_NE_mirrored_target.png
conditions/stimulus_manifest.csv
conditions/trial_sequence.csv
conditions/all_candidate_images.csv
conditions/unmatched_images.csv
conditions/manual_actor_selection_template.csv
logs/build_summary.json
```

If your `NimStim.zip` unzips somewhere else, either move it so the final path is `resources/NimStim/`, or pass the path explicitly:

```bash
./.conda_env/bin/python build_stimuli.py --nimstim-root /path/to/NimStim --seed 20260625
```

## Run The Experiment

Run windowed for local checking:

```bash
./run_with_env.sh --windowed
```

Run fullscreen for data collection:

```bash
./run_with_env.sh
```

The launcher uses the project-local conda environment at `.conda_env/` and writes PsychoPy preference/cache files under `.home/`.

PsychoPy should be run from a normal macOS Terminal session attached to the display. In headless shells, PsychoPy/pyglet can fail while looking for a screen even when the environment is installed correctly.

## Task Design

The task has 80 trials: 20 fearful, 20 angry, 20 happy, and 20 neutral targets. Each trial is:

```text
800 ms fixation cross
33 ms target face
467 ms neutral mask
7.7 s response screen
```

The response screen shows:

```text
1 = happy
2 = angry
3 = neutral
4 = fearful
```

The selected response is marked with a small circle until the next fixation cross starts.

The current configured actor set is:

```text
01F, 07F, 09F, 13F, 18F, 32M, 34M, 37M, 39M, 40M
```

This version uses closed-mouth NimStim variants only: `AN_C`, `FE_C`, `HA_C`, and `NE_C`.

For neutral target trials, the target is a left-right mirrored copy of that actor's neutral face, while the mask is the unmirrored neutral face from the same actor.

## Trial Order

Runtime does not reshuffle trials. The experiment reads the fixed order from:

```text
conditions/trial_sequence.csv
```

That file is generated by `build_stimuli.py` with seed `20260625`. The generated sequence has:

```text
80 total trials
20 trials per emotion
8 trials per actor
2 trials per actor/emotion combination
no immediate same-actor repetitions
no more than two identical emotion labels in a row
```

## Outputs

Participant CSV files are saved locally under:

```text
data/
```

Run logs and build summaries are saved locally under:

```text
logs/
```

Each trial row includes participant ID, session, actor ID, target emotion, target and mask paths, phase onsets, response key, response emotion, correctness, RT from target onset, RT from response-window onset, and trial duration.

## Response Mapping

The default response mapping is defined in `config.py`:

```text
1 = happy
2 = angry
3 = neutral
4 = fearful
```

You can override it at runtime:

```bash
./run_with_env.sh --response-map "1=fearful,2=neutral,3=angry,4=happy"
```

The response mapping used is saved in every data row.

## Replication Notes

This is a close conceptual replication, not an exact stimulus replication, because NimStim replaces the Ekman and Friesen face set used in the paper. The task timing, masking structure, number of identities, number of trials, target conditions, same-identity neutral masks, and mirrored-neutral target logic are matched as closely as possible.

Stimulus identities, facial morphology, image acquisition, and image properties may differ from the original Ekman and Friesen stimuli. Any publication, thesis, or preregistration should explicitly report the dataset substitution and deterministic selection policy used here.
